#Parse

Instead of modifying the native Date.prototype , persianDate.js creates a wrapper for the Date object. To get this wrapper object, simply call persianDate() with one of the supported input types.

#Now

persianDate();

To get the current date and time, just call persianDate() with no parameters. 

var now = persianDate();

This is essentially the same as calling persianDate(new Date()) .

#Unix Offset (milliseconds)

persianDate(Number);

Similar to new Date(Number) , you can create a moment by passing an integer value representing the number of milliseconds since the Unix Epoch (Jan 1 1970 12AM UTC). 

var day = persianDate(1318781876406);

#Date

persianDate(Date);

You can create a persianDate with a pre-existing native Javascript Date object. 

var day = new Date(2011, 9, 16);
var dayWrapper = persianDate(day);

This is the fastest way to get a persianDate.js wrapper.

#Array

persianDate(Number[]);

You can create a moment with an array of numbers that mirror the parameters passed to new Date()But As Persian Date Number Like [1393,2,22,11,22,30] 

[year, month, day, hour, minute, second, millisecond]
persianDate([1393, 1, 14, 15, 25, 50, 125]); // February 14th, 3:25:50.125 PM

Any value past the year is optional, and will default to the lowest possible number. 

persianDate([2010]); // January 1st
persianDate([2010, 6]); // July 1st
persianDate([2010, 6, 10]); // July 10th

Construction with an array will create a date in the current timezone. To create a date from an array at UTC, use persianDate.utc(Number[]) . 

persianDate.utc([2010, 1, 14, 15, 25, 50, 125]);

Note: Because this mirrors the native Date parameters, the following parameters are all zero indexed: months, hours, minutes, seconds, and milliseconds. Years and days of the month are 1 indexed.

This is often the cause of frustration, especially with months, so take note!

If the date represented by the array does not exist, persianDate#isValid will return false. 

persianDate([2010, 13]).isValid(); // false (not a real month)
persianDate([2010, 11, 31]).isValid(); // false (not a real day)
persianDate([2010, 2, 29]).isValid(); // false (not a leap year)

#PesianDate Clone

persianDate(Moment);

All moments are mutable. If you want a clone of a moment, you can do so explicitly or implicitly.

Calling persianDate() on a moment will clone it. 

var a = persianDate([2012]);
var b = persianDate(a);
a.year(2000);
b.year(); // 2012

var a = persianDate([2012]);
var b = a.clone();
a.year(2000);
b.year(); // 2012

#UTC1.5.0

persianDate.utc();
persianDate.utc(Number);
persianDate.utc(String);
persianDate.utc(String, String);
persianDate.utc(String, String[]);
persianDate.utc(String, String, String);
persianDate.utc(persianDate);
persianDate.utc(Date);

By default, moment parses and displays in local time.

If you want to parse or display a moment in UTC, you can use persianDate.utc() instead of persianDate() .

This brings us to an interesting feature of persianDate.js. UTC mode.

While in UTC mode, all display methods will display in UTC time instead of local time. 

persianDate().format();     // 2013-02-04T10:35:24-08:00
persianDate.utc().format(); // 2013-02-04T18:35:24+00:00

Additionally, while in UTC mode, all getters and setters will internally use the Date#getUTC* and Date#setUTC* methods instead of the Date#get* and Date#set* methods. 

persianDate.utc().seconds(30) === new Date().setUTCSeconds(30);
persianDate.utc().seconds()   === new Date().getUTCSeconds();

It is important to note that though the displays differ above, they are both the same moment in time. 

var a = persianDate();
var b = persianDate.utc();
a.format();  // 2013-02-04T10:35:24-08:00
b.format();  // 2013-02-04T18:35:24+00:00
a.valueOf(); // 1360002924000
b.valueOf(); // 1360002924000

Any moment created with persianDate.utc() will be in UTC mode, and any moment created with persianDate() will not.

To switch from UTC to local time, you can use persianDate#utc or persianDate#local. 

var a = persianDate.utc([2011, 0, 1, 8]);
a.hours(); // 8 UTC
a.local();
a.hours(); // 0 PST

#Get + Set

persainDate.js uses overloaded getters and setters. You may be familiar with this pattern from it's use in jQuery.

Calling these methods without parameters acts as a getter, and calling them with a parameter acts as a setter.

These map to the corresponding function on the native Date object. 

persianDate().seconds(30) === new Date().setSeconds(30);
persianDate().seconds()   === new Date().getSeconds();

If you are in UTC mode, they will map to the UTC equivalent. 

persianDate.utc().seconds(30) === new Date().setUTCSeconds(30);
persianDate.utc().seconds()   === new Date().getUTCSeconds();

For convenience, both singular and plural method names exist as of version 2.0.0 .

Note: All of these methods mutate the original moment when used as setters.

persianDate().milliseconds(); // Number

Gets or sets the milliseconds.

Accepts numbers from 0 to 999. If the range is exceeded, it will bubble up to the seconds.

#Date of Month

persianDate().date(Number);
persianDate().date(); // Number
persianDate().dates(Number);
persianDate().dates(); // Number

Gets or sets the day of the month.

Accepts numbers from 1 to 31. If the range is exceeded, it will bubble up to the months.

Note: persianDate#date is for the date of the month, and persianDate#day is for the day of the week.

#Day of Week

persianDate().day(Number);
persianDate().day(); // Number
persianDate().days(Number);
persianDate().days(); // Number

Gets or sets the day of the week.

This method can be used to set the day of the week, with Sunday as 0 and Saturday as 6.

If the range is exceeded, it will bubble up to other weeks. 

persianDate().day(-7); // last Sunday (0 - 7)
persianDate().day(7); // next Sunday (0 + 7)
persianDate().day(10); // next Wednesday (3 + 7)
persianDate().day(24); // 3 Wednesdays from now (3 + 7 + 7 + 7)

Note: persianDate#date is for the date of the month, and persianDate#day is for the day of the week.

#Year

persianDate().year(Number);
persianDate().year(); // Number
persianDate().years(Number);
persianDate().years(); // Number

Gets or sets the year.

Accepts numbers from -270,000 to 270,000.

#Manipulate

Once you have a Moment , you may want to manipulate it in some way. There are a number of methods to help with this.

persianDate.js uses the fluent interface pattern, also known as method chaining. This allows you to do crazy things like the following. 

persianDate().add('days', 7).subtract('months', 1).year(2009).hours(0).minutes(0).seconds(0);

Note: It should be noted that moments are mutable. Calling any of the manipulation methods will change the original persianDate.

If you want to create a copy and manipulate it, you should use persianDate#clone before manipulating the persianDate.More info on cloning.

#Add

persianDate().add(String, Number);
persianDate().add(Number, String); // 2.0.0
persianDate().add(Duration); // 1.6.0
persianDate().add(Object);

Mutates the original moment by adding time.

This is a pretty robust function for adding time to an existing persianDate. To add time, pass the key of what time you want to add, and the amount you want to add. 

persianDate().add('days', 7);

There are some shorthand keys as well if you're into that whole brevity thing. 

persianDate().add('d', 7);
Key Shorthand
years y
months M
weeks w
days d
hours h
minutes m
seconds s
milliseconds ms

If you want to add multiple different keys at the same time, you can pass them in as an object literal. 

persianDate().add('days', 7).add('months', 1); // with chaining
persianDate().add({days:7,months:1}); // with object literal

There are no upper limits for the amounts, so you can overload any of the parameters. 

persianDate().add('milliseconds', 1000000); // a million milliseconds
persianDate().add('days', 360); // 360 days

Special considerations for months and years

If the day of the month on the original date is greater than the number of days in the final month,
the day of the month will change to the last day in the final month. 

persianDate([2010, 0, 31]);                  // January 31
persianDate([2010, 0, 31]).add('months', 1); // February 28

There are also special considerations to keep in mind when adding time that crosses over Daylight Savings Time.
If you are adding years, months, weeks, or days, the original hour will always match the added hour. 

var m = persianDate(new Date(2011, 2, 12, 5, 0, 0)); // the day before DST in the US
m.hours(); // 5
m.add('days', 1).hours(); // 5

If you are adding hours, minutes, seconds, or milliseconds, the assumption is that you want precision to the hour, and will result in a different hour. 

var m = persianDate(new Date(2011, 2, 12, 5, 0, 0)); // the day before DST in the US
m.hours(); // 5
m.add('hours', 24).hours(); // 6

Alternatively, you can use durations to add to moments. 

var duration = persianDate.duration({'days' : 1});
persianDate([2012, 0, 31]).add(duration); // February 1

As of version 2.0.0, a reversed syntax is also supported to ease development. The syntaxes below will all work. 

persianDate().add('seconds', 1);
persianDate().add('seconds', '1');
persianDate().add(1, 'seconds');

This syntax will not work. The first parameter would need to be a number, not a string. 

persianDate().add('1', 'seconds');

#Start of Time

persianDate().startOf(String);

Mutates the original moment by setting it to the start of a unit of time. 

persianDate().startOf('year');   // set to January 1st, 12:00 am this year
persianDate().startOf('month');  // set to the first of this month, 12:00 am
persianDate().startOf('week');   // set to the first day of this week, 12:00 am
persianDate().startOf('day');    // set to 12:00 am today
persianDate().startOf('hour');   // set to now, but with 0 mins, 0 secs, and 0 ms
persianDate().startOf('minute'); // set to now, but with 0 seconds and 0 milliseconds
persianDate().startOf('second'); // same as persianDate().milliseconds(0);

These shortcuts are essentially the same as the following. 

persianDate().startOf('year');
persianDate().month(0).date(1).hours(0).minutes(0).seconds(0).milliseconds(0);

persianDate().startOf('hour');
persianDate().minutes(0).seconds(0).milliseconds(0)

As of version 2.0.0, persianDate#startOf('day') replaced persianDate#sod .

#End of Time

persianDate().endOf(String);

Mutates the original moment by setting it to the end of a unit of time.

This is the same as persianDate#startOf , only instead of setting to the start of a unit of time, it sets to the end of a unit of time. 

persianDate().endOf("year"); // set the moment to 12-31 11:59:59.999 pm this year

persianDate#eod Can Use Instead: persianDate#endOf('day') replaced .

#Local

persianDate().local();

Toggles a flag on the original moment to internally use Date#get* and Date#set* instead of Date#getUTC* and Date#setUTC* . 

var a = persianDate.utc([2011, 0, 1, 8]);
a.hours(); // 8 UTC
a.local();
a.hours(); // 0 PST

See persianDate.utc() for more information on UTC mode.

#UTC

persianDate().utc();

Toggles a flag on the original moment to internally use Date#getUTC* and Date#setUTC* instead of Date#get* and Date#set* . 

var a = persianDate([2011, 0, 1, 8]);
a.hours(); // 8 PST
a.utc();
a.hours(); // 16 UTC

See persianDate.utc() for more information on UTC mode.

Escaping characters

To escape characters in format strings, you can wrap the characters in square brackets. 

persianDate().format('[today] DDDD'); // 'today Sunday'

persianDate().fromNow();

#Time from X1.0.0

persianDate().from(Moment|String|Number|Date|Array);
persianDate().from(Moment|String|Number|Date|Array, Boolean);

You may want to display a moment in relation to a time other than now. In that case, you can use persianDate#from . 

var a = persianDate([2007, 0, 29]);
var b = persianDate([2007, 0, 28]);
a.from(b) // "a day ago"

The first parameter is anything you can pass to persianDate() or an actual Moment . 

var a = persianDate([2007, 0, 29]);
var b = persianDate([2007, 0, 28]);
a.from(b);                     // "a day ago"
a.from([2007, 0, 28]);         // "a day ago"
a.from(new Date(2007, 0, 28)); // "a day ago"
a.from("1-28-2007");           // "a day ago"

Like persianDate#fromNow , passing true as the second parameter returns value without the suffix. This is useful wherever you need to have a human readable length of time. 

var start = persianDate([2007, 0, 5]);
var end = persianDate([2007, 0, 10]);
start.from(end);       // "in 5 days"
start.from(end, true); // "5 days"

#Difference1.0.0

persianDate().diff(Moment|String|Number|Date|Array);
persianDate().diff(Moment|String|Number|Date|Array, String);
persianDate().diff(Moment|String|Number|Date|Array, String, Boolean);

To get the difference in milliseconds, use persianDate#diff like you would use persianDate#from . 

var a = persianDate([2007, 0, 29]);
var b = persianDate([2007, 0, 28]);
a.diff(b) // 86400000

To get the difference in another unit of measurement, pass that measurement as the second argument. 

var a = persianDate([2007, 0, 29]);
var b = persianDate([2007, 0, 28]);
a.diff(b, 'days') // 1

The supported measurements are years, months, weeks, days, hours, minutes, and seconds. For ease of development, the singular forms are supported as of 2.0.0. Units of measurement other than milliseconds are available in version 1.1.1.

By default, persianDate#diff will return number rounded down. If you want the floating point number, pass true as the third argument. Before 2.0.0, persianDate#diff returned rounded number, not a rounded down number. 

var a = persianDate([2007, 0]);
var b = persianDate([2008, 5]);
a.diff(b, 'years')       // 1
a.diff(b, 'years', true) // 1.5

If the moment is later than the moment you are passing to persianDate.fn.diff , the return value will be negative. 

var a = persianDate();
var b = persianDate().add('seconds', 1);
a.diff(b) // -1000
b.diff(a) // 1000

A easy way to think of this is by replacing .diff( with a minus operator. 

          // a < b
a.diff(b) // a - b < 0
b.diff(a) // b - a < 0

Month and year diffs

persianDate#diff has some special handling for month and year diffs. It is optimized to ensure that two months with the same date are always a whole number apart.

So Jan 15 to Feb 15 should be exactly 1 month.

Feb 28 to Mar 28 should be exactly 1 month.

Feb 28 2011 to Feb 28 2012 should be exactly 1 year.

See more discussion on the month and year diffs here

This change to month and year diffs was made in 2.0.0.

persianDate#unix outputs a Unix timestamp (the of seconds since the Unix Epoch).

This value is floored to the nearest second, and does not include a milliseconds component.

#As Javascript Date1.0.0

persianDate().toDate();

To get the native Date object that persianDate.js wraps, use persianDate#toDate .

This will return the Date that the moment uses, so any changes to that Date will cause the moment to change. If you want a Date that is a copy, use persianDate#clone before you use persianDate#toDate .

persianDate#native has been replaced by persianDate#toDate and has been deprecated as of 1.6.0.

#Is Leap Year

persianDate().isLeapYear();

persianDate#isLeapYear returns true if that year is a leap year, and false if it is not. 

persianDate([2000]).isLeapYear() // true
persianDate([2001]).isLeapYear() // false
persianDate([2100]).isLeapYear() // false

#Is Daylight Saving Time

persianDate#isDST checks if the current moment is in daylight savings time. 

persianDate([2011, 2, 12]).isDST(); // false, March 12 2011 is not DST
persianDate([2011, 2, 14]).isDST(); // true, March 14 2011 is DST
